Projet S2

Pokemon TCG

Développement d'un jeu de cartes à collectionner en ligne, mettant l'accent sur l'architecture logicielle et la communication client-serveur.

Contexte du Projet

La Situation d'Apprentissage et d'Évaluation (SAE) Pokemon TCG constitue un projet académique structuré en deux phases complémentaires. L'objectif était de maîtriser les deux versants du développement web (Backend & Frontend) à travers une approche modulaire.

Le Défi Majeur

S'adapter à une base de code existante (développée par une autre équipe) et assurer l'interopérabilité des systèmes.

Analyse Technique

1 Backend (PHP)

Conception d'une API RESTful complète. Modélisation de la base de données relationnelle pour gérer les decks, les cartes et les parties. Application stricte des principes SOLID et utilisation du pattern MVC.

2 Frontend (JS)

Création d'une interface dynamique sans rechargement de page (SPA like). Gestion des états de jeu complexes et des animations de cartes via JavaScript vanilla et des appels Fetch asynchrones.

Aperçu du Résultat

Capture d'écran de l'interface

Vue du plateau de jeu lors d'une partie active
