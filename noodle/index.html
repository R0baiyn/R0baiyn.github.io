Noodle (Audit & Refactoring)

Analyse, audit et refactoring complet d'une application PHP existante présentant de nombreuses failles de sécurité et un code source de mauvaise qualité.

📋 Contexte

Projet d'analyse, d'audit et de refactoring complet d'une application existante (Noodle) présentant un code source de mauvaise qualité et une base de données mal conçue. L'objectif était d'identifier tous les problèmes (sécurité, performance, architecture, qualité de code) puis de proposer et implémenter des solutions pour rendre l'application sûre, maintenable et optimisée.

📅
Dates
2026, Semestre 4
⏱️
Durée
~3 mois
🎯
Objectifs
Auditer une application existante, identifier les failles de sécurité (OWASP), corriger l'architecture (SOLID) et optimiser la base de données
⚠️
Contraintes
Code legacy sans documentation, base de données non normalisée, respect strict des recommandations OWASP

🚀 Démarche & Implication

Analyse approfondie du code source et de la base de données. Rédaction d'un rapport d'audit complet. Implémentation des corrections : refactoring du code avec application des principes SOLID, normalisation de la base de données (3FN), correction des failles de sécurité (injections SQL, XSS, CSRF). Travail méthodique d'analyse puis de correction en groupe de trois.
